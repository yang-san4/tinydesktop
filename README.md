# TinyDesktop

ブラウザ上で動くレトロ風デスクトップ OS シミュレーター。HTML5 Canvas + バニラ JavaScript のみで構築されており、外部ライブラリは一切使用していません。

440×330px の仮想スクリーン上に CRT モニター風の外観、BIOS ブートシーケンス、ウィンドウマネージャー、18 のアプリケーションを再現しています。WebGL の 3D ゲーム3本（FPS と三人称忍者アクション2作）、11サイトからなる偽インターネット、生態系つきアクアリウムまで、すべて単一ファイルのバニラ JS アプリとして動きます。

## スクリーンショット

> `index.html` をブラウザで開くだけで起動します。

## 機能

### デスクトップシステム

- BIOS 風ブートアニメーション
- ドラッグ＆リサイズ可能なウィンドウマネージャー
- タスクバー（ウィンドウ一覧 + システムクロック）
- スタートメニュー（アプリランチャー + テーマ切替）
- 右クリックコンテキストメニュー
- デスクトップアイコンのドラッグ配置
- デスクトップペット
- トースト通知システム
- サウンド ON/OFF（localStorage で永続化）

### 5 つのテーマ

Vapor / Wafuu / Mokume / Classic / Aqua — CSS カスタムプロパティによるリアルタイム切替（リロード不要）

## 使い方

```bash
# ローカルで開く（サーバー不要）
open tinydesktop/index.html

# または任意の HTTP サーバーで配信
npx serve tinydesktop
```

### キーボード操作

**GEKKO（FPS）**
- `WASD` — 移動
- マウス — 視点操作（Pointer Lock）
- クリック — 射撃
- `Space` — ジャンプ（二段ジャンプ対応）
- `Q` — ダッシュ
- `E` — グラップルフック
- `1` `2` `3` — 武器切替

**KAGE（忍者ARPG）**
- `WASD` — 移動 / `Shift` — 忍び歩き（足音を消す）
- マウス — カメラ（Pointer Lock）
- クリック — 斬撃（3段コンボ。非警戒の敵の背後なら闇討ち）
- `F` / 右クリック — 手裏剣（提灯を消して闇を作れる）
- `Space` — ジャンプ（二段対応）
- `Q` — ダッシュ（回避無敵）
- `E` — 鉤縄の構え（主観照準・時間減速）→ クリックで発射

**KAGE II（KAGE から操作変更）**
- `Shift` — 壁の近くで押すと壁張り付き、壁がなければしゃがみ（`C` は廃止）
- `W`/`A`/`S`/`D` を2回連打 — その方向へステップダッシュ（`Q` は廃止）
- しゃがみ中に方向2連打 — その方向へ回避ロール（前転・後転・側転、無敵つき）
- 壁張り付き中に方向2連打 — 壁沿いステップ。壁から離れる向きは壁を蹴って離脱、壁の方向へはステップ不可
- 張り付き中は被発見距離と足音が大幅減。`Space` で壁から飛び離れ、クリックで物陰から斬りかかる
- 屋内の赤茶の床板は「うぐいす張り」— 忍び歩き（`Shift`）以外で踏むと鳴いて警戒される

**Tetris**
- `←` `→` — 左右移動
- `↓` — ソフトドロップ
- `↑` / `X` — 右回転
- `Z` — 左回転
- `Space` — ハードドロップ

**Piano**
- `Z X C V B N M` — 白鍵（C D E F G A B）
- `S D` `G H J` — 黒鍵

**Terminal**
- `ls`, `cd`, `cat`, `grep`, `echo`, `mkdir`, `rm` など 27 コマンド対応
- パイプ `|`、リダイレクト `>` `>>`、`&&` チェーン、`$HOME` 変数展開

## 技術的な特徴

- **WebGL ネオンエンジン** — GEKKO v2 は低解像度レンダリング + ピクセル拡大でレトロな質感を保ちつつ、太陽光 + 半球環境光 + 動的ポイントライト、距離フォグ、プロシージャル空シェーダを実装（GEKKO Classic は Canvas 2D ソフトウェアラスタライザ版）
- **レイキャスティングエンジン** — 3D Maze は Wolfenstein 方式のレイキャスティングで擬似 3D を実現
- **固定タイムステップ物理** — Pinball はフレームレート非依存の安定したシミュレーション
- **疑似シェル** — Terminal はトークナイザ・パイプ・リダイレクト・ファイルシステムを約 1,400 行で実装
- **三人称ステルスエンジン** — KAGE は同エンジンを三人称化。衝突クリップ付き追従カメラ、視覚コーン + 3D 聴覚 + 提灯連動の3段階警戒 AI、ヒットストップ、フリーエイム鉤縄（着弾面の天面高さを読んで縁にマントリング）を実装
- **屋内レベルと壁張り付き** — KAGE II は床スラブの分割で階段吹き抜けを持つ多層屋内（天守4フロア）を構築。壁張り付きはオービットカメラの注視点を角の先へずらすだけで「覗き見」を実現し、うぐいす張りの床は警戒 AI への音イベント注入として実装
- **プロシージャルマテリアルと月光シャドウ** — KAGE II の描画コアは材質ID付き頂点 + シェーダ生成テクスチャ（漆喰・木目・瓦・石垣・畳・水面）と 1024² PCF シャドウマッピング、ブルーム+フィルミックグレードのポストパイプラインを実装
- **敵 AI とセルフテスト** — GEKKO / KAGE ともポーズ画面の AI TEST モードでゲーム自身が自動プレイ。KAGE はタイトルからボス撃破までの完全クリアをヘッドレス Chrome ハーネスで回帰検証している
- **偽インターネットの生態系** — TinyWeb は日付シードの日替わりコンテンツ、localStorage 永続化（ゲストブック・視聴数・いいね・ハイスコア）、サイト間を巡る Webring とスパム導線まで再現
- **テーマシステム** — CSS カスタムプロパティで全アプリがリアルタイムにテーマ追従

## アーキテクチャ

各アプリは **IIFE（即時実行関数式）** で独立したスコープに隔離されています。グローバル名前空間の汚染を防ぎ、アプリ間の干渉を排除しています。

ウィンドウの開閉・最小化は `MutationObserver` で監視し、非表示時にゲームループを自動停止することでリソースを節約しています。

描画は基本的に Canvas 2D API、GEKKO v2 / KAGE / KAGE II は WebGL（自作の箱ジオメトリエンジン、1つの箱定義が描画と衝突判定の両方を生成）。いずれも低解像度レンダリング + `image-rendering: pixelated` でレトロなピクセルアート表現に統一しています。

ゲームの検証はヘッドレス Chrome + requestAnimationFrame スタブのハーネスで行い、ゲーム内蔵の AI TEST モードに自動プレイさせて回帰を検出します。
