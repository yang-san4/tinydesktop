# KAGE II — AAA級アップグレード（壁張り付き・4倍ステージ・天守内部）

2026年6月12日。KAGE のエンジンをベースに、続編「KAGE II」を `js/kage2.js`（約3,400行）として新規追加した。既存の KAGE はそのまま残し、デスクトップに両方が並ぶ。テーマは3つ — 新システム「壁張り付き」、約4倍規模のステージ、そして御殿・天守の**屋内潜入**。

## 1. 壁張り付き + コーナー覗き見（新システム）

C キーで壁に張り付き、A/D で壁沿いにスライド。壁の端に達してさらに押し込むと「覗き見」状態になり、カメラが角の先へ回り込む。張り付き中は被発見距離が大幅に縮み（×0.32、覗き見中は×0.55）、移動音も消える。

```js
function updateCling(dt){
  // カメラ相対入力を壁の接線へ射影
  var tx=-player.clingNy,ty=player.clingNx;
  var lat=mx*tx+my*ty;            // 壁沿い成分
  var away=mx*player.clingNx+...; // 壁から離れる成分 → 0.18秒で解除
  if(Math.abs(lat)>0.25){
    var nx2=player.x+tx*step,ny2=player.y+ty*step;
    if(wallBehindAt(nx2,ny2)&&...){ player.x=nx2;player.y=ny2; }   // 壁が続く限り滑る
    else player.peek=lerp(player.peek,dirS,6*dt);                  // 壁が切れた=角 → 覗く
  }
}
```

カメラは覗き見量 `peek` に応じて注視点を角の先へ平行移動するだけで、既存の `cameraClip`（壁めり込み防止）がそのまま面倒を見てくれる。

```js
if(player.cling){
  lookX+=tx*player.peek*1.6+player.clingNx*0.35;  // 角の先へ視点を送り出す
  lookY+=ty*player.peek*1.6+player.clingNy*0.35;
}
```

**学習ポイント**: 「覗き見カメラ」は専用のカメラモードを書かずに、オービットカメラの**注視点をずらすだけ**で成立する。衝突処理（cameraClip）を注視点ベースで設計しておくと、後から来る演出系の要求をタダで吸収できる。

## 2. ステージ4倍化と屋内レベルの構築

3ゾーン→4ゾーン（外町 SOTOMACHI / 大手門 OTEMON / 二の丸御殿 NINOMARU / 天守 TENSHU）。御殿は中央廊下+4部屋の屋内、天守は**4フロアの内部**（1F物置→2F番所→3F武器庫→4F御座の間）を階段で登る。

屋内は「箱=描画+衝突」の統一ジオメトリをそのまま使う。床スラブに**階段吹き抜けの穴**を開けるには、スラブを複数の箱に分割する:

```js
// 2F床 = 中央+外周の庇 + 吹き抜け脇の2片（x[-6.6,-4.6]×y[9.5,14] が穴）
S(1.65,8.5,5.7,12.5,14.8,0.4,...);   // 中央〜東（庇含む）
S(-7.25,8.5,5.7,1.3,14.8,0.4,...);   // 西外周の庇
S(-5.6,5.3,5.7,2.0,8.4,0.4,...);     // 穴の南側
S(-5.6,14.95,5.7,2.0,1.9,0.4,...);   // 穴の北側
stairsY(-5.6,13.0,3,6,-1,1.8);       // 0.5刻みの段差 → step-up 物理で歩いて登れる
```

### うぐいす張り（鳴く床）

廊下に赤茶の床板を仕込み、忍び歩き（SHIFT）より速く踏むと「キリ…」と鳴いて周囲9uの敵に警戒が走る。

```js
if(hs2>SNEAK_SPD+0.3&&Math.abs(player.x-sb.x)<sb.w/2&&...){
  sb.cd=0.7; playSound('squeak');
  noisePing(player.x,player.y,player.z,9);  // susp+=0.7、2回踏むと警報
}
```

**学習ポイント**: 「音を立てる床」は当たり判定+クールダウン+既存の警戒システムへの注入だけで作れる。**新メカニクスは新システムを作るより、既存システム（警戒AI）への入力を1本増やす**方が安くて強い。

## 3. 捜索AI（最終目撃地点）

警戒した敵はプレイヤーが見えなくなると**最後に見た地点**まで掃討に行き、着いたら周囲を見回す。`e.lastSeen` を視認時に更新し、見失ったら追跡対象を lastSeen に切り替えるだけ。横丁に隠れる→敵が通り過ぎる、という忍者らしい駆け引きが生まれる。

## 4. AI TEST の収束保証（回帰テストの設計）

ステージが複雑になると、AI の詰みパターンが激増した。実際に踏んだ罠と対策:

| 詰み | 対策 |
|---|---|
| 塀がメインストリートを塞ぐ | レベル側を修正（通り沿い→通りの西側へ） |
| 階段の中腹に南から衝突→暴発ジャンプで壁越え脱出 | 階段位置を調整 + ルートに回り込みウェイポイント |
| 階段最上段が壁天端より高く飛び越え可能 | 階段脇の外壁を上階まで連続させる |
| ボス戦死亡→復活で wp0 に戻り3階で永久壁押し | **復活時は現在地に最も近い wp から再開**（z重み付き） |
| 25秒進捗なし | ウォッチドッグで地蔵へ戻して再走 |

```js

  for(var i=0;i<route.length;i++){
    var d=dist2d(player.x,player.y,w.x,w.y)+Math.abs((w.z||0)-player.z)*4; // zは4倍重み
    ...
  }
}
```

最終結果: **タイトル→4ゾーン→殿様撃破まで AI が自力完走（190〜260秒）**。これが KAGE II の回帰テストになる。

**学習ポイント**: 複雑なレベルでの自動走行は「絶対に詰まない」より「**詰んでも必ず復帰できる**」設計（進捗ウォッチドッグ + 最寄り wp 再開）の方が堅牢。死亡・落下・押し出しの全てが「少し戻ってやり直し」に正規化される。

## 5. その他の品質改善

- **カメラ近接時の自機非表示**: 屋内でカメラが背中に密着すると画面が自機で塞がる → クリップ距離 <1.6 で自機を描画しない（FPS視点に自然に移行）
- ボスの起動条件に z 差チェック（床越しに目覚めない）
- 敵が階段吹き抜けから降ってくる（ledge guard の許容落差内）— 意図しない挙動だが「上の階の敵が降りてくる」として採用
- 罠の多い裏手にも地面を敷き、理不尽な即死を排除

## 6. グラフィック AAA パス（同日フォローアップ）

「もっとリアルでグラフィカルにシネマティックに、ただしタイニー OS の世界観の中で」という要望を受けた第2弾。低解像度（400×300→拡大）とピクセル質感は堅持したまま、光と空気を作り直した。

### ピクセル単位ライティング

頂点ライティングをフラグメントシェーダへ移行。大きな壁の1枚クワッドにも提灯の光だまりがグラデーションで落ちる。

- 動的ライト 4→6灯（行灯の多い屋内が映える）
- 月光スペキュラ（屋根瓦に月のシーン）
- 高さ重み付きフォグ（霧は地表に沈み、天守の上階は澄む）
- ワールド座標の量子化ハッシュで**ピクセルグレイン**（`hash(floor(vPos*7.0))`）— テクスチャなしで面の単調さを消し、ドット絵の質感はむしろ強まる

### ブルーム + フィルミックグレード（ポストパイプライン）

シーンを FBO に描き、輝度抽出（200×150）→ 2パスガウスぼかし → 合成。合成シェーダで S カーブコントラスト + シャドウを青く、ハイライトを暖かく分離トーン。提灯・月・斬撃トレイル・敵の警戒眼が滲んで光る。

```glsl
c = mix(c, c*c*(3.0-2.0*c), 0.22);        // gentle filmic S-curve
c += vec3(0.012,0.018,0.045)*(1.0-l);     // cool lift in the shadows
c += vec3(0.050,0.024,0.0)*l*l;           // warm bloom in the highlights
```

### 空・大気・シネマ演出

- 空シェーダに**流れる雲**（2オクターブ値ノイズ）、大きめの満月、遠景の山稜シルエット
- 蛍（運河・庭園・桜の周りを漂う加算ビルボード）
- シネマティック**レターボックス**（ゾーン告知・鉤縄照準・ボス登場時に上下黒帯）
- **ボス登場カット**: スローモー + カメラが殿様へプッシュ + タイトルカード「— 殿 THE LORD —」
- 微細な**フィルムグレイン**（128×96 ノイズキャンバスを毎フレームランダムオフセットで α0.05 合成）

### ハマったポイントと学習

1. **近接面の白飛び**: ピクセル単位ライティングにしたら提灯至近の面が飽和して緑がかった白に。→ ソフト Reinhard（0.75 超のみ圧縮）で色相を保ったままロールオフ
2. **グロースプライトの罠**: 提灯にグロー用ビルボードを足したら、至近距離で「光る正方形」に見えた。ブルームが既にハローを作るので**グロースプライトは冗長**——削除が正解
3. **「光る謎の巨大キューブ」の正体**: 覗き見カメラが通りの提灯の 1u 手前に引いた結果、発光ボックスがドアップになっていただけ。**カメラから 1.4u 以内の提灯は描画スキップ**（光源は維持）で解決。デバッグはカメラ座標をテストフックで吐いてスクリーン座標から逆算した

**学習ポイント**: ポスト処理を入れると「今まで許されていた素材の粗」が一斉に可視化される（emissive の強さ、ライト強度、カメラとプロップの距離）。描画品質のアップグレードは描画だけで閉じず、**ライティング値とカメラ運用の再調整までがセット**。

## 7. キャラクターモデルの関節リグ化（同日フォローアップ第3弾）

「3Dモデルをリアルにしたい」を受けて、全キャラクターを単純な箱の積み上げから**関節のあるリグ**に作り直した。1体あたりの箱数は約12→25〜30。

### 新プリミティブ: テーパー付きボックス

`dynBoxT` — 上面・下面のスケールを別々に指定できる箱。これだけで和装・甲冑の特徴的なシルエットが一気に出る。

```js
dynBoxT(cx,cy,cz,hx,hy,hz,yaw,pitch,roll,col,em,tsT,tsB)
// tsT=0.16 → 陣笠の円錐 / tsT=0.62 → 兜の鉢
// tsB=1.45 → 錣（しころ）の裾広がり / tsB=1.25 → 草摺・袴のフレア
```

### 関節システム

`limbSeg`（ピボットからぶら下がる1節）を連結して2関節肢を構成。`legPair` が太もも+すね+足の歩行サイクルを生成し、**膝はスイングの戻りで曲がる**（`kB=max(0,sin(ph-1.2))*amp*1.4`）。歩行位相は実際に移動した距離から駆動するので、速度が変わっても足が滑らない。

```js
e.walkPh += dist2d(now, last) * 6.5;  // 移動量→歩行位相（フットスライド防止）
```

### 各モデルの見どころ

- **忍者（自機）**: 肩へ向けて広がる胴のテーパー、膝の曲がる走り、空中では脚をたたむジャンプポーズ、忍び歩きで深くかがむ。背中の鞘と柄、4節のスカーフ
- **足軽**: 陣笠（円錐）、胴当て+草摺4枚、脚絆、直立に担いだ槍（警戒すると腰だめに構え直す）、提灯持ちは左手にぶら下げる
- **弓兵**: 上下に反った3節の弓+弦、矢筒に矢2本、警戒時は左腕で弓を突き出し右腕で引き絞る
- **侍/精鋭**: 兜の鉢+錣+前立て、面頬、大袖、胴の威し（金ライン2本）、草摺5枚、佩刀は手の位置に追従
- **殿様**: 裾広がりの袴、金の陣羽織（第2形態で脱ぐ）、大兜+三日月前立て、大太刀

### ハマったポイント

武器の長軸を「ローカル y + pitch+90°」で書いたら、構え角度の符号が直感と合わず槍が地面を引きずった。**長軸はローカル z（直立基準）に統一**し、pitch だけで前傾を表す規約に直して解決。リグを書くときは「全部品で軸の規約を1つに固定する」のが先決。
