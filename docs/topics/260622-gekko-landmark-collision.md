# GEKKO ランドマークの当たり判定追加
<!-- updated: 2026-06-22 -->

## 状況
ファンタジーFPS（GEKKO, `js/fps.js`）に後から追加した木・井戸・山・城・風車に当たり判定がなく、すり抜けていた。これらのランドマークの本体部分に当たり判定を付与し、プレイヤーが衝突・乗り上げできるようにした。葉・屋根・水面など装飾部分は引き続き非衝突のまま。

## 経緯
### 2026-06-22
#### ランドマークに当たり判定を付与
- `js/fps.js` の当たり判定は `solids[]`（AABB のリスト）が駆動する統一ジオメトリ方式。`S()` は「可視＋衝突」、`D()` は「装飾のみ・非衝突」。両者は引数シグネチャが同一。
- 後付けランドマーク（`tree` / `well` / `windmill` / `castle` / `mountain`）は全パーツを `D()` で描いていたため衝突なし。本体パーツを `S()` に切り替えるだけで解決（色・寸法はそのまま）。
  - well: 石枠 `D→S`（上に乗れる）
  - windmill: 塔本体 `D→S`
  - castle: 中央天守 + 四隅の塔を `D→S`（中庭は開いたまま歩ける）
  - mountain: 下2段の岩タイルを `D→S`（段差を登れる。雪冠は装飾のまま）
- 木だけは幹が細い（0.24）ため `D→S` だと「1.3 幅の葉の下を 0.24 の見えない箱で塞ぐ」違和感が出る。見た目と独立した衝突専用ヘルパ `COL()` を新設し、幹付近に girth 0.55*scale・高さ 1.0*scale の不可視ソリッドを置いた。

  ```js
  // 衝突専用 AABB（不可視。見た目と衝突範囲がずれるランドマーク用）
  function COL(cx,cy,z0,w,d,h){
    solids.push({x:cx,y:cy,z0:z0,top:z0+h,w:w,d:d});
  }
  // tree:
  COL(cx,cy,z,0.55*s,0.55*s,1.0*s); // 細い視覚的な幹とは別に、樹冠の下が固く感じる太さで塞ぐ
  ```
  **学習ポイント**: `S`/`D` のシグネチャを揃えてあるおかげで「衝突を付ける＝D を S に変えるだけ」になっている。これは v1 で「描画用リストと衝突用リストを分けたら『見えるのに当たらない』バグが多発した」反省から来た統一ジオメトリ設計（コード冒頭コメント参照）。見た目と衝突範囲を意図的にずらしたいケース（細い幹）だけ `COL()` で例外的に分離する、という切り分けが綺麗。
- ヘッダコメント `MEDIEVAL LANDMARKS (decor only — no collision...)` は実態と矛盾するため「本体は衝突／葉・屋根・水面は装飾」に更新。
- 構文チェック: ローカルに node/deno/bun が無く実行確認は未。変更は同一シグネチャの `D→S` 置換と単純なヘルパ追加のみで、目視で確認済み。
