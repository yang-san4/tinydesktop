## 経緯
### 2026-06-19
#### マップを縦3層 + 南北拡張に全面再設計
ユーザー要望「マップをデカく、縦に広いほうが楽しい、上下に広げて移動範囲を広く」。確認の結果、高さ(Z)も南北(Y)も両方拡張／既存レイアウト全面再設計／バランス調整込みと決定。

**設計の物理的根拠（コードから逆算）** — これが層間隔・島間隔を決める唯一の根拠:
- ジェット打ち上げ ≈ +7〜8m（`JET_VZMAX×JET_MAX`＋ジャンプ初速）→ **層間隔6〜7mに固定**
- グラップル射程16m直線（上向きほど水平到達減）→ **隣接足場の縁どうしは16m未満（実用8〜12）**
- 着地は「XY重なりsolidの最高top」に吸われる（fps.js:972-1004）→ **層の中心を2〜4mずらして真上重ねを回避**し、縁に下層へ降りる開口を作る
- 飛行敵ホバーは`player.z`相対（fps.js:1228）なので高くしても敵AIは破綻しない

**buildWorld 構成（3層）**:
- 下層 top -2〜0: 中央ハブ`island(0,0,0,20,20)`＋南北東西の広い回遊フロア（南は-34へ拡張）＋橋4本
- 中層 top 6〜8: ハブ`island(2,2,7,16,16)`（中心+2ずらし）＋4島＋ツインタワー＋キャットウォーク
- 上層 top 13〜16: ハブ`island(0,-2,14,11,11)`（さらにずらし）＋3島＋十字カバー柱＋ボス足場
- 垂直導線: グラップル石11個を縦螺旋に（下層→中層→上層）
- 遠景skyTower 15→10本に削減、配置半径を60〜80へ外出し

**定数変更**（fps.js:148/151/153/156）:
```
FAR 120→160, FOG_START 28→38, FOG_END 68→95   (広く高くなった分の描画/フォグ)
JET_MAX 1.2→1.5, JET_VZMAX 5.0→5.8            (層間6〜7mを1回で登り切るマージン)
GRAP_RNG 16→18                                 (縦移動の保険、設計は16基準)
VOID_Z -7→-10                                  (下層top-2帯で即死多発を防ぐ)
```
**学習ポイント**: マップの「広さ」はビジュアルでなく**プレイヤーの到達力（ジェット燃料×上昇速度、グラップル射程）から逆算**して決めるのが肝。先に層間隔を到達力に合わせて固定し、定数はそれを快適にする微調整に留めた。着地が「最高top吸い込み」仕様なので、層を縦に積むときは中心オフセットで開口を作らないと下層へ降りられなくなる——これが3D縦マップ最大の落とし穴。

- SPAWN_PTS を8→12点に拡張し3層へ分散（各点をisland topに一致。startWaveが飛行+2.5/地上+0.5を加算）
- プレイヤー初期/リスポーンは原点(0,0,0.5)維持で変更なし（下層ハブが原点）

#### ジャンプ調整（デフォルト高さUP + 押しっぱなしで頂点ジェット）
縦マップ化後の追加要望。2点とも `js/fps.js` 内の小変更。
- デフォルトジャンプ: `JUMP_VEL 6.5→8.0`（到達 約1.17m→約1.78m、+50%）。ダブルジャンプは`JUMP_VEL*0.85`なので自動追従。その後、実機の体感で `JUMP_VEL 8.0→9.0`（到達 約2.25m）・`JET_MAX 1.5→1.8`（ジェット燃料上限+20%）に再調整。
- 押しっぱなしで頂点ジェット起動: jetCutoff 解除条件を追加。
  ```js
  if(!keys.sp)jetCutoff=false;
  else if(jetCutoff&&!player.grounded&&player.vz<=0)jetCutoff=false; // 押しっぱなし: 頂点でジェット自動起動
  ```
  **学習ポイント**: ジェット起動はもともと `jetCutoff` フラグでジャンプ上昇中の暴発を抑えていたが、解除がスペース解放(`!keys.sp`)のみだったため「押しっぱなしでは頂点で出ない＝離して押し直し必須」だった（コメントの "after jump apex-ish" は意図だけで未実装だった）。`vz<=0`（頂点通過）で解除を1行足すだけで、上昇→頂点→ジェットが滑らかに繋がる。ダブルジャンプは押下エッジ`spJustPressed`依存なので押しっぱなしでは暴発しない＝この変更と共存できるのがポイント。

#### 床抜けバグ修正（高速落下着地時のすり抜け）
ジャンプ高さUP＋ジェット強化＋縦マップで落下高度が増え「ジャンプ＆ジェット後の着地でたまに床を抜ける」症状。原因を特定して1行修正。
- 原因: `resolvePhysics` の処理順が「積分 → XY衝突押し出し → Z着地判定」。`dt`は最大0.05sクランプ。上層(top14)からの落下は約22m/sに達し、dt=0.05で**1フレーム約1.1m沈む**。着地フレームで足が天面を大きく貫通（`feet < s.top-0.01`）すると、XY押し出しの除外条件 `if(feet>=s.top-0.01||head<=s.z0)continue;` をすり抜け、**着地すべき足場を「壁」とみなして横に弾き出す**。その後のZ着地判定は狭いfootprint(`×0.7`)で重なりを見るため、弾かれた位置では着地に失敗→床抜け。
- 修正: XY押し出しループに1行追加。
  ```js
  if(player.vz<=0&&prevZ>=s.top-0.2)continue; // 降下中にこの天面へ着地中: 壁でなく床
  ```
  **学習ポイント**: 条件 `prevZ>=s.top-0.2` は直後のZ着地判定の縦条件と**完全一致**させてある。「このフレームにZ着地でスナップされ得る足場」をXY押し出しでは壁扱いしない、という整合。`prevZ`はフレーム開始時の高さなので、横から壁にぶつかるケース（prevZが天面より下）はガードに掛からず従来どおり壁として機能＝退行しない。**衝突解決は「順序」と「同じ対象に複数の判定が別の閾値で触る」のが事故の元**。離散時間ステップ＋分離した押し出しは高速移動でトンネリングするのが定石的な落とし穴で、判定間で閾値を揃えるのが効く。

#### スタイル刷新（SKYHOLM）土台フェーズ
夕暮れネオン→晴れた青空の中世風空中都市。ゲーム性（衝突・座標・敵・移動）は不変、見た目だけ。
- **空**: スカイシェーダ(FS_SKY)はJSの`SKY_TOP/HOR/BOT`をGLSL文字列に埋め込む実装。定数を青空（天頂[0.22,0.48,0.85]/地平[0.80,0.90,0.98]）に。太陽円盤は白め・小さく、グロー控えめ。**星ブロック(h>0.12)は昼なので削除**（`hash`/`uTime`は未使用化するがGLSL上は無害）。
- **光**: `SUN_DIR`を頭上高く、`SUN_COL`温白[1.25,1.18,1.02]、`AMB_UP`空青/`AMB_DN`地面暖色、`FOG_C`青白[0.78,0.86,0.95]（clearColorも追従）。
- **色/脱ネオン**: パレット`C_*`を石/木/草/瓦に。`edgeTrim`の`emissive 1→0`、`bridge`欄干・`building`窓のemissiveを0/0.3に下げ、`lamp`はランタン(小さな暖色発光)、`crate`は木。
- **新規`roof()`**: `pushQuad`で四方の斜面を棟点に寄せた切妻/寄棟屋根（装飾のみ＝衝突なし）。`building`と`skyTower`(山頂)で使用。
- **遠景**: `skyTower`をネオンビル→段重ねテーパーの山/浮遊島シルエット（茶岩＋草、高いものは雪冠）に書き換え。
- **雲**: 16→28個・白(1,1,1)・高め・ゆっくり。
- **UI**: ネオンCSS色を一括置換（`#00e5ff`→金茶`#c89b5a`、`#ff0060`→濃木`#8a4a2a`、`#c8d8e8`→羊皮紙`#e8dcc4`、`#0ae`→`#b5894f`、rgbaの青/桃/暗紫背景も茶系へ）。危険赤`#ff3c3c`とHP状態色は意味色なので維持。
- **改名**: タイトル画面・WebGL非対応文言・index.htmlの表示ラベルを SKYHOLM に。
  **学習ポイント**: WebGLの色は頂点カラー(RGB)＋α=emissive。**αを0にすると法線ライティングに従い、1だと自己発光**。ネオン感の正体はこのemissive=1なので、形状を保ったままemisscoiveを0に落とすだけでマットな実在感の質感に変わる。空シェーダは定数をGLSL文字列にビルド時連結しているので、JS定数を変えるだけで再コンパイル不要で空色が変わる（uniform化されていない点に注意＝動的に変えるならuniform化が必要）。UI配色はCSS文字列の色トークンを一括置換するのが最速だが、意味を持つ色（危険=赤、HP残量のグラデ）は機械置換の対象から外す判断が要る。
