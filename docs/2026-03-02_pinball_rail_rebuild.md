# Haunted Pinball — レール式プランジャーレーン全面リビルド

**日付**: 2026-03-02
**ファイル**: `js/pinball.js`（2284行、全面書き換え）

---

## 変更概要

壁ベースのプランジャーレーン物理が120x210pxキャンバスで根本的に機能しなかったため、**レール＋ステートマシン方式**で全面リビルド。

## 核心的変更: レール式プランジャーレーン

### Before（壊れていた方式）
```javascript
// 壁セグメントで10px幅の通路を構成
{ x1: 68, y1: 55, x2: 68, y2: 140 },  // plunger lane left wall
{ x1: 78, y1: 5,  x2: 78, y2: 145 },  // plunger lane right wall
{ x1: 78, y1: 145, x2: 65, y2: 150 }, // lane top deflector

// 射出後すぐに state = 'playing' → 壁物理で制御
state = 'playing';
```
**問題**: 10px幅の通路でボール(r=2.5)を壁反射で制御するのは不安定。ボールが壁を貫通したり挟まったりする。

### After（レール方式）
```javascript
// ステートマシンに 'in_lane' を追加
// title → ready → launching → in_lane → playing → draining/ballsave → ready/gameover

if (state === 'in_lane') {
  ballVY -= GRAVITY;
  ballX = LANE_X;  // X座標を73に固定（水平移動不可）
  ballY += ballVY;  // 垂直方向のみ移動

  if (ballY >= 140) {
    // レーン上端到達 → フィールドに遷移
    state = 'playing';
    ballX = 64;       // フィールド右端に配置
    ballVX = -2.0;    // 左カーブでフィールドに入る
    ballVY *= 0.7;    // 速度減衰
  }

  if (ballY < 8) {
    // パワー不足 → ready に戻す
    state = 'ready';
    resetBall();
  }
}
```

## 学習ポイント

### 1. 狭い通路の物理シミュレーションは避ける
120x210pxのような小さなキャンバスでは、壁で構成した狭い通路にボールを通すのは非常に困難。ボール半径と通路幅の比率が大きすぎると、数値誤差やサブステップの粗さでボールが壁を貫通する。

**解決策**: 物理的な壁ではなく、ステートマシンで制御する。ボールの座標を直接制限することで、物理演算の不安定さを回避。

### 2. ステートマシンの分離
```
in_lane: X固定、Y方向のみ、壁判定なし
playing: 自由物理、壁判定あり
```
状態ごとに物理ルールを完全に分けることで、各状態の動作が明確になり、デバッグしやすくなる。

### 3. フィールドガードパターン
```javascript
// ボールがレーンエリアに再侵入するのを防ぐ
if (ballX > 66 && ballY < 140) {
  ballX = 66;
  ballVX = -Math.abs(ballVX) * 0.5;
}
```
壁の物理判定ではなく、座標クランプで再侵入を防ぐ。シンプルで確実。

### 4. 壁定義の簡素化
レーン壁を削除し、フィールドのみの5本に簡素化:
```javascript
var WALLS = [
  { x1: 2,  y1: 20,  x2: 2,  y2: 170 },   // 左壁
  { x1: 66, y1: 55,  x2: 66, y2: 170 },   // 右壁
  { x1: 2,  y1: 170, x2: 66, y2: 170 },   // 上壁
  { x1: 2,  y1: 20,  x2: 15, y2: 5 },     // 左ドレインガイド
  { x1: 66, y1: 55,  x2: 55, y2: 5 }      // 右ドレインガイド
];
```

### 5. 描画の分離
`drawPlungerLane()` でレーンの視覚表現（背景、壁の線、プランジャー、レーン内ボール）を一括描画。`in_lane` 状態のボールもこの関数内で描画し、`drawBall3D()` は `playing` 状態専用。

## 射出パラメータ

```javascript
ballVY = power * 5.0 + 3.5  // power: 0〜1
// 最小速度: 3.5（パワー0%）
// 最大速度: 8.5（パワー100%）
// MAX_VEL = 7.0（クランプ適用）
// 約35%パワーでレーン通過可能
```
